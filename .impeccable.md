# Design Context

## Users

Regular AI enthusiasts who want to input API keys, base URLs, and change ports to get started. Advanced settings should be hidden by default, waiting for power users to discover. Use case: local development and debugging of AI service routing, quickly configuring multi-provider environments.

## Brand Personality

**Keywords**: Professional restraint, deceptively simple, depth in details. Not a consumer product—this is a professional control panel that appears simple on the surface but reveals advanced capabilities in the details. Avoid flashy or over-decorated aesthetics. Pursue a "seems simple but is actually powerful" character.

## Aesthetic Direction

**Visual tone**: The current warm gradient + glassmorphism direction is correct, but UI elements are cluttered and messy. Need to simplify layout and reduce visual noise. Maintain professionalism. Use HeroUI design tokens for automatic dark mode adaptation—no separate implementation needed.

**Positive references**: Linear (clean and restrained), Vercel dashboard (professional without being flashy), GitHub Settings (functional clarity without clutter)

**Anti-references**: Notion (too consumer-focused), Figma landing (too marketing-driven), typical AI slop interfaces with hero metrics and card grids

## Design Principles

1. **Simple surface, powerful depth** - Show only essential fields by default, collapse advanced options
2. **Restrained decoration** - Reduce gradient layers, simplify glassmorphism, lower visual complexity
3. **Function-first layout** - Fix cluttered element stacking, establish clear visual hierarchy
4. **Professional tool character** - Don't pursue consumer product aesthetics, aim for practical developer control panel beauty
5. **Design token driven** - Use HeroUI tokens throughout, avoid hard-coded colors, ensure theme consistency
